# AI-агент для автоматизации проверки документов

Этот проект представляет собой AI-агент для автоматизации проверки комплектности документов и целевого использования льготных кредитов (АПК). Решение заменяет медленный процесс ручного переноса данных из документов в Excel, обеспечивая высокую точность и контроль качества.

## 1. Как запустить проект
Убедитесь, что у вас установлен Python 3.11+.

*   **Установка зависимостей:**
    ```bash
    pip install -r requirements.txt
    ```
*   **Запуск полного пайплайна (тестирование):**
    ```bash
    PYTHONPATH=. pytest -v
    ```
*   **Запуск проверки тестовых предметов оплаты (subjects_test.txt):**
    ```bash
    PYTHONPATH=. python run_subject.py
    ```

## 2. Использованные технологии и обоснование
*   **Python 3.11+**: Основной язык разработки.
*   **LangChain**: Фреймворк для управления LLM-цепочками и промптами. Выбран для обеспечения гибкости при интеграции различных языковых моделей.
*   **Pydantic**: Используется для валидации структуры данных, возвращаемых нейросетью, что гарантирует строгость формата JSON.
*   **Pytest**: Фреймворк для обеспечения надежности пайплайна через модульное тестирование.
*   **RegEx (re)**: Модуль для высокоскоростного извлечения данных из неструктурированных текстов, позволяющий работать без тяжелых ML-моделей на этапе парсинга.

## 4. Компромиссы
* Обработка "неуверенности": Для файла scan_ocr_001.txt, который является намеренно сложным (низкое качество OCR), система не пытается "угадать" данные. Функция extract() честно возвращает None для полей, которые не удалось распознать, что является признаком качественной обработки неуверенности.  


* RegEx vs NER: Использование регулярных выражений вместо NER-моделей обеспечивает высокую скорость работы и отсутствие необходимости обучения тяжелых моделей

* Fallback-логика: Оффлайн-метод проверки имеет 100% специфичность (не пропускает нецелевые траты), но его чувствительность ограничена списком ключевых слов

## 5. Как проверить работу сервиса
Запуск команды PYTHONPATH=. pytest -v выполняет 25 проверок:

Валидация форматов сумм и дат.

Корректность классификации типов документов.

Валидация работы логики check_subject на всех тестовых файлах.
## 6. Примеры вызова функций
    from src.extractor import extract
    from src.check_subject import check_subject

    # Извлечение данных
    text = "Сумма: 1 250 000,00 руб. Предмет: Карбамид марки Б."
    print(extract(text))
    # {'amount': 1250000.0, 'subject': 'Карбамид марки Б.'}

    # Проверка предмета
    matches, conf, reason = check_subject("Карбамид марки Б")
    print(f"Match: {matches}, Reason: {reason}")
    # Match: True, Reason: [Fallback] Обнаружены маркеры сельскохозяйственной номенклатуры.
